Wiki 目录结构（Obsidian Vault）
wiki/
├── INDEX.md ← 全局入口 + 统计
├── TOPIC-MAP.md ← 研究主题层次
├── papers/
│   ├── INDEX.md ← 按年分组
│   └── {arxiv_id}.md ← 中文深度分析
├── concepts/
│   ├── INDEX.md ← 字母排序
│   └── {slug}.md ← 跨论文概念综合
└── raw/ ← 不可变原始层
    └── papers/{arxiv_id}/
       ├── meta.yaml ← 结构化元数据
       ├── fulltext.md ← 论文全文
       └── repo-readme.md ← GitHub README
核心思想：LLM 是无状态的 → Wiki 是 LLM 的外部大脑。
每次 Q&A 对话从 INDEX.md 出发导航 wiki，获取上下文后回答。
Obsidian Graph View 显示论文-概念的知识图谱连接。

复合循环: 论文 → wiki → Q&A → 洞见回填 → 丰富未来编译

6 编排层 — 两个入口
daily_pipeline.py — 每日自动
1. 多源搜索 (arXiv + S2 + RSS) → 候选池
2. 去重 (seen_papers.json)
3. 三级漏斗 → ScoredPaper[]
4. 反馈环 (corpus + stats + wiki)
5. 飞书推送 + Notion/Zotero/Git 同步
python3 -m scripts.daily_pipeline
cold_start_force_vla.py — 批量冷启动
1. 搜索真实论文 (search_force_vla.py)
2. 批量 ingest → wiki/raw/ (网络 I/O)
3. 批量 V2 compile → wiki/papers + concepts (LLM)
4. 重建全部索引
python3 -m scripts.cold_start_force_vla --batch 5
